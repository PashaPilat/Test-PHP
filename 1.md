# Проект «Категории и товары»
## 🎯 Задача
Новый проект на чистом PHP. Цель — реализовать страницу каталога товаров и категорий (пример: demo.solomono.net).
### Функциональные требования:
- Создать таблицы товаров и категорий в базе данных, связать их отношением «один‑ко‑многим».
- Заполнить БД тестовыми данными: 10–20 товаров (с датой, ценой, названием) и 3–5 категорий.
- Вывести страницу:
    - слева список категорий с количеством товаров в каждой;
    - справа список товаров.
- При клике на категорию — AJAX‑обновление списка товаров справа.
- Добавить селектбокс сортировки: «дешевле», «по алфавиту», «новые».
- При выборе сортировки — обновлять список товаров без перезагрузки.
- У каждого товара кнопка «Купить» → модальное окно Bootstrap с информацией о товаре.
**Обязательно:**
- использовать ООП;
- чистый PHP (без фреймворков);
- сортировка и фильтрация без перезагрузки страницы;
- передача данных PHP → JS в формате JSON.

**Дополнительно:**
- при смене категории/сортировки добавлять GET‑параметры в URL, чтобы при перезагрузке оставаться на выбранной категории/сортировке.

### ⚙️ Архитектура билдера
Билдер (`src/DB/Select/Builder.php`) реализует fluent‑интерфейс для построения SQL‑запросов.
Он разбит на отдельные классы по частям SQL:

- **Where** `(Tools/Where.php)` — условия фильтрации.
- **Join** `(Select/Join.php)` — соединения таблиц.
- **GroupBy / Having** — группировка и агрегаты.
- **OrderBy** — сортировка.
- **Limit / Offset** — постраничная выборка.

Для строгой типизации используются enum `(в папке Tools/Enums)`:
- **JoinType** — `INNER, LEFT, RIGHT, FULL`.
- **JoinOperator** — `EQ, NEQ, GT, LT` и т.д.
- **WhereOperator** — `EQ, LIKE, IN` и т.д.
- **Boolean** — `AND, OR`.
- **OrderDirection** — `ASC, DESC`.

#### 📖 **Примеры использования**
Простая выборка
```php
$rows = DB::select('id', 'name')
    ->from('products')
    ->where('status', 'active')
    ->orderBy('price', OrderDirection::ASC)
    ->all();
```    
**SQL:**
```sql
SELECT id, name FROM products WHERE status = ? ORDER BY price ASC
JOIN 
```
```php
$rows = DB::select('*')
    ->from('categories')
    ->join('products', 'categories.id', 'products.category_id', JoinOperator::EQ, JoinType::INNER)
    ->all();
```    
**SQL:**
```sql
SELECT * FROM categories
INNER JOIN products ON categories.id = products.category_id
```
**Пагинация**
```php
$pageRows = DB::select('*')
    ->from('products')
    ->where('status', 'active')
    ->paginate(2, 10); // страница 2, по 10 строк
```    
**SQL:**
```sql
SELECT * FROM products WHERE status = ? LIMIT 10 OFFSET 10
HAVING
```
```php
$rows = DB::select('category_id, COUNT(*) as cnt')
    ->from('products')
    ->groupBy('category_id')
    ->having('COUNT(*)', 5, WhereOperator::GT)
    ->all();
```    
**SQL:**
```sql
SELECT category_id, COUNT(*) as cnt
FROM products
GROUP BY category_id
HAVING COUNT(*) > ?
```

### 📂 Categories

Таблица категорий товаров. Поддерживает **древовидную структуру** (иерархию категорий).

| Поле | Тип | Описание |
|-----|-----|-----|
| id | int | Уникальный идентификатор категории |
| name | varchar(255) | Название категории |
| slug | varchar(255) | URL slug |
| path | varchar(255) | Путь категории от корня |
| parent_id | int | Родительская категория |
| description | text | Описание категории |
| icon | varchar(255) | Путь к иконке категории |
| status | enum(active,inactive) | Статус категории |
| created_at | timestamp | Дата создания |
| updated_at | timestamp | Дата обновления |

#### ⚙ Особенности

- Категории образуют **иерархическое дерево**
- Корневая категория имеет `id = 1`
- Поле `path` хранит путь в дереве (`1/2/5`)

#### 📑 Индексы

```
idx_category_status
idx_category_slug
idx_category_parent
```

### 📊 Counters

Таблица агрегированных счетчиков для оптимизации запросов.

Используется для быстрого получения:

- количества дочерних категорий
- количества товаров в категории
- количества товаров с атрибутом
- количества вариантов значений атрибута

| Поле | Тип | Описание |
|-----|-----|-----|
| entity_type | enum(category,attribute) | Тип сущности |
| entity_id | int | ID сущности |
| children_count | int | Количество дочерних категорий |
| products_count | int | Количество товаров |
| variants_count | int | Количество вариантов значений |

#### 📑 Индексы

```
UNIQUE(entity_type, entity_id)
INDEX(entity_type, entity_id)
```

## 👁 Database Views

### 📂 categories_with_count

Представление категорий с количеством:

- дочерних категорий
- товаров

```
SELECT
    c.*,
    children_count,
    products_count
FROM categories c
LEFT JOIN counters cnt
ON cnt.entity_type = 'category'
AND cnt.entity_id = c.id
```

---

### 🧩 attributes_with_count

Представление атрибутов с количеством:

- товаров
- вариантов значений

```
SELECT
    a.*,
    products_count,
    variants_count
FROM attributes a
LEFT JOIN counters cnt
ON cnt.entity_type = 'attribute'
AND cnt.entity_id = a.id
```

## categories_with_count

```sql
SELECT
    c.*,
    children_count,
    products_count
FROM categories c
LEFT JOIN counters cnt
ON cnt.entity_type = 'category'
AND cnt.entity_id = c.id
```
